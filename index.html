ğŸ”„ Instruction Cycle

The instruction cycle is the complete process of fetching and executing an instruction.

  • Fetch Cycle: Instruction is fetched from memory using the PC and loaded into IR.
  • Execute Cycle: Instruction is decoded and executed by the processor.

Cycle repeats until HALT instruction.

Bus Interconnection

Bus Interconnection

A bus interconnection is a common communication path used to connect the CPU, main memory, and I/O devices. It allows the transfer of data, instructions, and control signals using shared signal lines. Only one device can control the bus at a time. Shared buses reduce hardware cost but may cause performance bottlenecks. Modern systems often use point-to-point interconnections.

ğŸ” RECAP: INTERRUPTS

1. Meaning of Interrupt

An interrupt is a signal sent to the CPU to request immediate attention.

  • Generated by I/O devices
  • Generated by internal CPU conditions

The CPU pauses the current program (does not discard it).

MCQ Focus: Interrupt is NOT a program, instruction, or interpreter.

2. Purpose of Interrupts

Interrupts improve CPU efficiency.

  • Without interrupts â†’ CPU waits idle during I/O
  • With interrupts â†’ CPU executes another program

MCQ Focus: Interrupts reduce wasted CPU time.

3. Interrupt Service Routine (ISR)

ISR is a special OS routine executed when an interrupt occurs.

  • Identifies interrupt source
  • Performs service
  • Restores CPU state
  • Resumes program

MCQ Focus: ISR is NOT a user program.

4. Program Execution Without Interrupts

CPU waits until I/O completes and remains idle.

MCQ Focus: CPU time is wasted.

5. Program Execution With Interrupts

CPU overlaps I/O and program execution.

MCQ Focus: Interrupts allow parallelism.

6. Instruction Cycle With Interrupts

  1. Fetch
  2. Execute
  3. Interrupt

MCQ Focus: Interrupt cycle is added to instruction cycle.

7. Steps in Interrupt Processing

  1. Complete current instruction
  2. Save PC and registers
  3. Disable interrupts
  4. Load ISR address
  5. Execute ISR
  6. Restore context
  7. Resume program

MCQ Focus: CPU always finishes current instruction first.

8. Context Saving

  • Program Counter
  • Registers
  • Processor state

MCQ Focus: Context saving prevents data loss.

9. Interrupt Vector

A table that stores addresses of ISRs.

MCQ Focus: Stores addresses, NOT instructions.

10. Types of Interrupts

External: Keyboard, printer, disk (I/O devices)

Internal: Divide by zero, overflow, invalid instruction

MCQ Focus: I/O â†’ External | Errors â†’ Internal

11. Multiple Interrupts

Handled using priority and masking.

MCQ Focus: Higher priority is serviced first.

12. Interrupt Priority

Determines the order of interrupt servicing.

13. Interrupt Enable & Disable

Used to protect critical sections.

MCQ Focus: Interrupts are disabled during ISR.

14. Nested Interrupts

Higher priority interrupts can interrupt an ISR.

MCQ Focus: Multiple context saves occur.
